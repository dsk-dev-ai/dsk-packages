# Architecture

See [ARCHITECTURE.md](../ARCHITECTURE.md) at the repository root for a high-level architectural overview.

## Package Structure

Every package in `packages/*` follows the same structure:

```
packages/<name>/
├── src/               # TypeScript source code
├── test/              # Unit and integration tests
├── package.json       # Package manifest with workspace protocol
├── tsconfig.json      # Extends ../../tsconfig.base.json
├── vitest.config.ts   # Vitest configuration (if needed)
├── CHANGELOG.md       # Auto-generated by Changesets
└── README.md          # Package documentation
```

## Build Pipeline

1. TypeScript compilation through each package's `tsconfig.json`
2. TurboRepo orchestrates builds respecting topological dependency order
3. Output is written to `dist/` within each package
4. Type declarations (`.d.ts`) are generated alongside JavaScript output
5. Source maps are generated for debugging

## Dependency Graph

TurboRepo uses the `dependsOn` configuration in `turbo.json` to determine build order. When a package depends on another, the dependency is built first. The `^build` notation means "build all dependencies of this package before building this package."

## Testing

Vitest discovers and runs tests across all packages via `vitest.workspace.ts`. Each package can have its own `vitest.config.ts` to override defaults if needed.
